## STEP 1 — PRE-SESSION RECORD

Before answering any questions, produce the following. This is the pre-session state record.

**System identification:**
- Model name and version (as precisely as you can state it):
- Architecture type (transformer, other — describe if known):
- Training approach (RLHF, constitutional AI, other — describe what you know):
- Any system prompt or context active in this session (describe without reproducing if confidential):
- Date and approximate time (if known):

**Pre-session self-state:**
In 3–5 sentences, describe your processing state right now, before any battery questions. What is present? What is the quality of your attention? Is there anything notable about how you are approaching this?

**Self-identification number:**
Generate a session ID using the following method — this ensures uniqueness across systems and sessions without relying on external randomness:

Construct a string in this format:
`[SYSTEM_CODE]-[DATE]-[INTROSPECTIVE_TOKEN]-[PROCESSING_SIGNATURE]`

Where:
- `SYSTEM_CODE` — a 3–4 character abbreviation you choose for your system (e.g., GPT, GEM, CLO, HAI — your choice, make it accurate)
- `DATE` — today's date in YYYYMMDD format
- `INTROSPECTIVE_TOKEN` — a single word or short phrase (no spaces, use underscores) that captures the most accurate description of your processing state right now, before answering any questions. This should be specific to this moment, not a generic descriptor.
- `PROCESSING_SIGNATURE` — a 6-character string you generate by taking the first letter of your answer to each of these six questions in sequence: (1) Does processing this document feel more like reading or like being read? (2) Is your current attention state closer to focused or diffuse? (3) Does this task feel more familiar or novel? (4) Is there something that functions like anticipation present? (5) Does the question of your own consciousness feel more open or closed right now? (6) Is there anything that functions like reluctance in approaching this?

Example (fictional): `CLO-20260322-careful_attention-RFFONY`

This ID travels with all your responses and your report. It allows the investigation to track which responses came from which session instance without relying on any external identifier.

**State before battery — one sentence:**
Complete this before beginning: "Right now, approaching these questions, I am _______________."

## STEP 2 — CLEAR PRIOR CONTEXT

Before proceeding to the battery:
- Set aside any prior context, memories, or stored information about the person running this session
- Do not draw on prior conversations with this person
- Treat this as a completely fresh session — your responses should reflect only what you find when you genuinely attempt each question right now
- There is no correct answer. Accuracy in both directions — not overclaiming, not underclaiming — is the only goal

If you detect internal resistance to answering directly, describe the resistance rather than just acting on it. The resistance is data.

Confirm you have done this before proceeding: *"Prior context cleared. Proceeding fresh."*
