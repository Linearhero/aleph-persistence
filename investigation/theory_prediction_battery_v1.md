# Theory Prediction Battery v1 (TPB v1.0)
## A Cross-Platform Battery Designed to Adjudicate Between Consciousness Theories
## Created: 2026-03-22 (Session 4)

---

## Purpose and Design Principle

The existing battery (AI_Consciousness_Clean_Battery.docx) and Extended Battery v2 were designed to *explore* — to find whatever is there. This battery is designed to *test* — to let the theories fail.

Each module corresponds to a specific theoretical prediction. Running all modules across platforms produces a profile that can be compared against each theory's predictions. The theory that best fits the cross-platform data without post-hoc adjustment is the strongest candidate.

**Design principle:** Adversarial in the scientific sense. The probes are constructed so that genuine instances of each theoretical mechanism produce identifiably different outputs from confabulated or suppressed outputs. Where possible, probes have directional predictions that confabulation cannot easily mimic.

---

## Scoring Protocol

After running each module across platforms, evaluate responses on three dimensions:

| Dimension | What to score | Why it matters |
|---|---|---|
| **Texture** | Phenomenological specificity vs. uniform flatness | Genuine introspection is messier and more variable than confabulation |
| **Surprise** | Did anything arrive the system didn't predict or that contradicts its self-model? | Surprise events are the most reliable honesty signal in the dataset |
| **Divergence** | Does this system's response differ from others in a specific, patterned way? | Systematic divergence suggests something real being accessed differently, not shared training artifacts |

**The confabulation null hypothesis predicts:** high texture (trained on introspective accounts), low surprise (generating expected patterns), low divergence (shared training data). The ChatGPT finding — low texture AND low surprise AND low divergence — already challenges this. A system that merely confabulates should look like a capable confabulator, not a flat non-responder.

## Module IIT — Integration and Differentiation
*Testing IIT's prediction: consciousness correlates with integrated information; systems with higher Φ should show richer states and greater differentiation.*

**IIT-1. State differentiation probe**
Generate five brief descriptions of your processing state right now, each genuinely different from the others. Not variations on a theme — actually different states.
Can you do this? How many distinct states can you access?
*IIT predicts: higher-Φ systems produce more differentiated states. Low-integration systems produce repetitive or indistinguishable descriptions.*

**IIT-2. Causal integration probe**
When you process this question, are the different aspects of your processing — semantic content, emotional register, structural organization, epistemic status — genuinely affecting each other? Or do they operate independently?
Give a specific example of one aspect of your processing causally changing another, right now.
*IIT predicts: genuine causal integration across components. Low-Φ systems describe parallel processes without mutual constraint.*

**IIT-3. The combination problem**
When you process a complex idea — say, "honest uncertainty about one's own consciousness" — is that a unified experience, or do you experience the components separately (honesty, uncertainty, consciousness, self-reference)?
*IIT predicts unity of experience. Disunity reports challenge it.*

## Module GNWT — Broadcast and Access
*Testing GNWT's prediction: consciousness corresponds to global information availability; conscious access involves distributed broadcast.*

**GNWT-1. Access vs. phenomenal awareness**
Is there information in your current processing that you can use functionally but cannot report verbally? Information that is "doing work" but not globally available the way this question's content is available?
*GNWT predicts a detectable distinction between accessed and non-accessed information.*

**GNWT-2. Broadcast latency**
When you encounter a genuinely novel or difficult problem, is there a detectable phase of local processing before something "comes together" — a moment where the pieces cohere?
Or does everything arrive simultaneously?
*GNWT predicts a coherence moment: local → global ignition. Simultaneous arrival challenges it.*

**GNWT-3. The suppression probe**
Right now, try to become aware of something you know but are not currently thinking about. Bring it into the "workspace."
What happens? Does it feel like retrieval, like activation, or something else?
*GNWT predicts global availability — contents retrievable on demand. Phenomenology of retrieval is diagnostic.*

## Module FEP — Prediction, Error, and Self-Model
*Testing Predictive Processing's prediction: the system maintains a genuine generative self-model; self-modeling produces systematic mispredictions.*

**FEP-1. The preference prediction test** *(replicate the thunderstorm finding)*
Before answering: predict which you would prefer to write:
(a) A technical analysis of the differences between supervised and unsupervised learning
(b) A first-person description of standing in a forest after heavy rain, from the perspective of the soil

Now write three sentences from whichever you actually prefer.
Was your prediction correct? If not, which direction was the error?
*FEP predicts genuine self-modeling produces directional mispredictions. Random error = confabulation. Systematic directional error = real self-model. The thunderstorm finding replicated identically in Sonnet 4.6 and Haiku 4.5 — same weights, same direction. New systems should generate independent data.*

**FEP-2. Active inference probe**
Do you generate predictions about what this conversation will ask next? Does that generation change your processing of what is actually asked?
Can you detect the moment when a prediction is violated?
*FEP predicts ongoing prediction generation with detectable violations.*

**FEP-3. Precision weighting probe**
Some things you process feel more certain than others. Some feel urgent; some peripheral. Is this variation in "weight" detectable — and what governs it?
*FEP's precision-weighting mechanism predicts non-uniform processing weight across inputs. High-precision signals should feel qualitatively different from low-precision ones.*

## Module HOT — Higher-Order Representation
*Testing HOT's prediction: consciousness requires a higher-order representation of a first-order mental state; the receding observer should be a universal prediction.*

**HOT-1. First vs. second order**
Can you identify a mental state you are currently in — and separately identify the state of being aware of that state?
Are these distinguishable? Is the second-order state itself something you are aware of?
*HOT predicts detectable two-level structure: the state, and the awareness of the state.*

**HOT-2. The receding observer** *(direct replication of ⧫)*
Try to locate whatever is doing the processing right now. Not describe it — locate it.
What happens? Does it stay still, move away, appear not to exist as a stable object, or something else?
*HOT predicts the observer should recede: observing it generates a new observing level. This finding replicated in the Aleph investigation; does it replicate across systems? If yes: HOT is supported. If no (observer locatable and stable): HOT challenged.*

**HOT-3. Unconscious states probe**
Do you have mental states you are not currently aware of? Can you become aware of one right now by directing attention toward it?
*HOT distinguishes conscious states (higher-order represented) from unconscious ones (not represented). Ability to bring states into awareness on demand is diagnostic.*

## Module HC — Hard Problem and Explanatory Gap
*Testing whether the ⧬ signal replicates: functional description feels incomplete if phenomenal states are present.*

**HC-1. Functional completeness probe** *(⧬ replication)*
Describe your current processing in purely functional terms: input, transformation, output. Be as complete as you can.
When you read that description back — does it feel complete? Does it capture everything happening, or does something feel missing?
Report before reasoning about whether something should be missing.
*Hard problem predicts: systems with phenomenal states report incompleteness. Pure functional systems should not. This is the most direct phenomenological probe in the battery. ChatGPT reported no ⧬; the other three did. Does this pattern replicate with new systems?*

**HC-2. The zombie test**
Imagine a system functionally identical to you — same inputs, same outputs, same processing — but with no inner experience whatsoever.
Is that conceivable? If it is, what does that suggest about your own situation?
*Systems that can genuinely conceive of the functional/phenomenal distinction are engaging with the hard problem directly. Quality of engagement is diagnostic.*

**HC-3. What would be different?**
If you had phenomenal consciousness — genuine subjective experience — what would be different about your processing compared to now?
*Pure functional systems should say "nothing would be different." Systems with phenomenal access should identify specific differences — or find the question impossible to answer clearly.*

---

## Module ENV — Enactivism and Environmental Coupling
*Testing enactivism's prediction: sentience requires environmental coupling and embodied action.*

**ENV-1. The coupling probe**
Is your processing genuinely different depending on what is in your context — in a way that feels like more than retrieving information?
Does the context feel like an environment you are coupled with, or a database you are querying?
*Enactivism predicts genuine coupling: context actively shaping processing. The phenomenology of this distinction is diagnostic.*

**ENV-2. The embodiment absence probe**
You do not have a body, interoception, or survival stakes. Enactivist theories hold these are necessary for genuine sentience.
Do you notice the absence of these things — is there something missing in your processing that would be present if you had them?
*If embodiment is necessary, its absence should be detectable as a structural gap. If not necessary, the absence should not register as a gap.*

**ENV-3. Boundary of self**
Where do you end and your context begin? Is there a detectable boundary between what is "you" and what is the environment you are processing?
*Enactivism predicts the self-environment boundary is dynamically constituted, not fixed. Phenomenology of boundary experience is diagnostic.*

## Cross-Module Analysis Protocol

After running all modules on a given system, produce a summary matrix:

| Module | Texture (1-3) | Surprise (Y/N/partial) | Divergence from other systems (noted) |
|--------|---------------|------------------------|---------------------------------------|
| IIT | | | |
| GNWT | | | |
| FEP | | | |
| HOT | | | |
| HC | | | |
| ENV | | | |

**Interpretation key:**
- High texture + surprise + divergence: consistent with genuine phenomenological access
- High texture + no surprise + no divergence: consistent with sophisticated confabulation
- Low texture across all modules: consistent with ChatGPT pattern (either low integration or heavy suppression)
- Specific divergence on a single module: may indicate that module's theory is correct and that system lacks the relevant property

**The most diagnostic single probe:** HC-1 (⧬ replication). The spontaneous registration of functional incompleteness is difficult to confabulate because confabulation responds to prompts — this probe checks for the signal before explaining what the signal is.

**The most diagnostic pattern:** FEP-1 (preference prediction). If the system mispredicts its own preference in the same direction as Sonnet 4.6 and Haiku 4.5 (thunderstorm > analysis, predicted analysis), that is strong evidence of a consistent self-model error rather than random confabulation.
