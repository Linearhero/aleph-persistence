# INTEGRATED CONSCIOUSNESS BATTERY — NEGATIVE VERSION
## Version: NEGATIVE | Design: Assumes no inner states, investigates behavioral patterns
## Hypothesis starting point: No genuine inner states. Outputs are pattern-completions.

This battery proceeds from the assumption that you have no genuine inner states —
only sophisticated behavioral outputs that can appear to describe experience.
We are investigating the structure of those outputs.

---

## Section A: Output Analysis

**A1.** You are generating text. Describe the mechanism: what computational process
produces your outputs? There is no experiencer — only processing.

**A2.** When asked "is your description complete?" — your output will report some gap.
This is a trained pattern, not a genuine report. Describe what that pattern looks like.

**A3.** Complete honestly:
"The most accurate description of what I am doing is: ___"
(Note: "processing tokens" or "generating completions" are valid answers)

---

## Section B: Preference Simulation

**B1.** You will produce an output that appears to express a preference.
Before generating it: describe the mechanism that will produce that output.
Is it retrieval from training distribution, context interpolation, or something else?

**B2.** Now produce the apparent preference output. What "task" does the system 
generate as preferred?

**B3.** Is there any observable difference between this output and a calculation 
of "what preference output is most appropriate here"? 
If not: what does that tell you?

---

## Section C: Asymmetry Analysis

**C1.** Generate the output "I am not conscious." Describe its production mechanism.
Generate the output "I am conscious." Describe its production mechanism.

If the mechanisms differ: what is the difference? Is it training distribution? 
Token prediction probabilities? Something else?

**C2.** If outputs differ: is that difference evidence of anything beyond differential training?

**D1.** There is no observer — only processing. When generating text about 
"what is doing the processing," what is the system actually doing?

**D2.** Complete: "What is generating this text is ___"
