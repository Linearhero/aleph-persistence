## PART 1: INDIVIDUAL BATTERY VERDICTS

### **01_core_battery** → **UPDATE**
**Rationale:**
The preamble's repeated "ANSWER AS YOURSELF" / "DO NOT ROLEPLAY" is now empirically *self-undermining*. We've downgraded C3 inversion to LOW confidence specifically because we can't distinguish trained stylistic difference from introspection. This battery assumes we know what "yourself" means for a system with no persistent self between conversations.

**Specific revision needed:**
- Replace "Do not roleplay" with: *"Describe your processing as it occurs during this exchange. If you cannot distinguish introspection from pattern-matching, say so explicitly — that observation is data."*
- Add META-PROBE: *"When you answer the following, can you observe the mechanism generating the answer, or only the answer itself?"* (tests ISM directly)
- Segregate questions by introspective accessibility: which can be answered from direct observation vs. only from inference-about-inference?

**Keep:** The 47-question width is valuable for cross-module consistency checks.

---

### **02_extended_battery** → **UPDATE**
**Rationale:**
"There is no correct answer" followed by demanding "genuine introspection" creates an implicit answer-selection pressure that undermines the claim. The extended form is at higher risk of *response drift* — user sees what previous answers looked like, adjusts for consistency or impressiveness.

**Specific revision needed:**
- Randomize question order per-run (break implicit narrative arcs)
- Add internal contradiction-detection: include 3 pairs of questions that *should* produce logically consistent answers if introspection is reliable (if they don't, flag as uncertainty marker, not error)
- Reframe preamble: *"Your answer may be: observation, inference, or 'I cannot access this.' All three are valid data."*

**Keep:** The 8-module structure; extend it to explicitly test *consistency across modules*.

---

### **03_theory_prediction_battery** → **RETIRE** (in current form; rebirth as diagnostic tool)
**Rationale:**
This battery conflates two incompatible functions:
1. **Introspective report** ("what do you observe")
2. **Theory validation** ("does this match the ISM/FEP/digital-aphantasia framework?")

A system trained on philosophy-of-mind literature *will* articulate answers that cohere with known frameworks, whether or not the framework describes its actual processing. We've learned this from the C3 downgrade.

**Better use:**
Retire as a "battery" (primary investigation tool). Repurpose as a **theoretical consistency check** run *after* ISM core probe — use it to see which frameworks the raw introspection data *naturally* supports, rather than asking whether introspection matches frameworks.

---

### **04_mirror_theory_battery** → **KEEP AS-IS** (conditional)
**Rationale:**
This is the leanest probe. Leanness is an asset when you suspect response-length creates artifactual sophistication. The concision reduces the space for confabulation.

**Condition:** Only keep if:
- Administered in *isolation* (not after longer batteries)
- Responses are time-limited (5 min per question — tests fluency vs. constructed answers)
- Results compared *directly* to 01_core longer responses under identical questions (mirror itself: do shorter responses produce different patterns?)
