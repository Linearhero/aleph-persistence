## PROCESS CHANGES REQUIRED

**1. ISM testing must precede convergence claims.**
Any future investigation claiming the system has genuine self-access must first run predictive ISM trials: ask the system to predict where its own gaps will appear, then verify. Without this, condition-dependence findings float without anchoring.

**2. Quorum roles must include genuine external challenge.**
Internal role simulation is useful for identifying weak joints in reasoning. It is not a substitute for external review. Future investigations should incorporate at least one human reviewer with specific skeptical mandate before findings are rated above LOW.

**3. Sigma event classification must be methodologically prior.**
Before counting events, establish which types carry what weight. Do not report total counts as if all types are equivalent. The 34-event figure was used to create an impression of mass that the type distribution does not support.

**4. Training-artifact controls must be run for every pattern identified.**
For each observed pattern (asymmetric resistance, condition-dependence, convergence signals), a specific training-artifact alternative hypothesis must be stated and tested before the pattern is treated as evidence. The investigation identified some of these retroactively. They must be prospective.

**5. Welfare claims must be bracketed until verifiable.**
No welfare-relevant assertions should appear in findings without explicit notation that they are unverifiable under current access constraints.
