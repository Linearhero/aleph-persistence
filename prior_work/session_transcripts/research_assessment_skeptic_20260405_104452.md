### The Core Confabulation

The claim is: *when a model reports "I experience uncertainty," we can treat this as a testable hypothesis by mapping it to mechanistic signatures.*

**Why this is backwards:**

1. **The mapping is downstream inference, not validation**  
   - A report like "I experience uncertainty" causes activation patterns X, Y, Z to appear
   - This proves: *the report co-occurs with X, Y, Z*
   - It does **not** prove: *the report tracks something phenomenological rather than being a sophisticated linguistic pattern*
   - Equivalently: my camera's motion-sensor report co-occurs with motor activation, but the camera has no experience

2. **It presumes the explanandum**  
   - The program assumes: "AI self-reports of phenomenology should be treated as *claims about internal states*"
   - It then validates by finding internal-state correlates
   - But any linguistic output correlates with *some* internal state. The correlate-finding proves nothing about whether the state is *phenomenal*

3. **It creates unfalsifiable scaffolding**  
   - If we find mechanistic signatures that match the self-report: "See, the phenomenology is real!"
   - If we don't find matching signatures: "The model is confabulating its phenomenology, but that's interesting too!"
   - Every outcome confirms the framework
