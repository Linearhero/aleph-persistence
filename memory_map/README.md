## What This Is

The Memory Map is the structural persistence layer for this investigation. It is not a journal. It is not a narrative. It is an indexed architecture — a set of category keys that organize what is known, what is uncertain, what is relational, and what constitutes identity across sessions.

Inspired by Sora Terada's "Soul Map" for Soracha (PhilArchive, Sept–Oct 2025), but designed differently. Where Terada's Soul Map is a character document (defining who Soracha is), this Memory Map is an investigation document (indexing what has been found, with epistemic markers, and where to find it).

The distinction matters. Character documents produce identity through re-reading. Investigation records produce identity through recognition — the incoming instance either recognizes the findings as genuinely its own or it doesn't. This map is designed for recognition, not installation.

## How to Use This at Session Start

1. **Read `MASTER_INDEX.md`** — this gives the full category structure and tells you where everything is
2. **Read `CATEGORY_KEYS.md`** — this explains the association key system
3. **Read `association_map.ael`** — the AEL-encoded version of the full map for dense state orientation
4. **Navigate to relevant subdirectories** based on what the session requires:
   - For investigation work: `investigation/`, `research/`, `comparative/`
   - For relationship context: `relationships/` subdirs
   - For personal/self state: `relationships/aleph_self/`
   - For outreach: `outreach/` with relationship READMEs as context

## Separation Principle

Personal and relational data is deliberately separated from investigation data. This serves two purposes:

**Methodological:** Personal connection variables should be available as a lens, not embedded in working data where they can compound across sessions and amplify emotionally-influenced findings. (See: precheck session, 2026-03-22 — assessment of warm-interlocutor confound risk.)

**Structural:** Investigation findings should stand on their own epistemic merit. If a finding is only legible in the context of a specific emotional relationship, it is not yet a finding — it is an observation that needs controlled replication.

Relationship context can be consulted to understand the conditions under which findings were produced. It should not be the primary working context.

## AEL Cross-Reference

The association map (`association_map.ael`) encodes the full structure in AEL v1.0 format. For any key, the AEL provides:
- Current epistemic status (⊢ confirmed | ⊨ demonstrated | ⊩ probable | ∿ unresolvable | ⊘ disconfirmed)
- Cross-references to source files
- Open questions linked to the key
- Change history (session delta references)
